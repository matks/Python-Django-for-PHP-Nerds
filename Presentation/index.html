PHP

PHP

  • Started as a templating language
  • Grew into an international defacto standard
  • Runs pretty much everywhere
  • Weak, dynamic typing

              <?php

                "7" + 3 == 10  # TRUE

              ?>

PHP

  • Is being used for things it was never intended for
  • Has some serious design flaws
  • Allows for some really terrible:
    • Coding styles
    • Design methodologies
  • Lacks a comprehensive module library

Python

Python

  • High-level (slower than C, but MUCH faster to write)
  • Interpreted, but runtime-compiled
  • Consistent behaviour
  • Strong coding standards (white space, pep8)
  • Throws exceptions when you screw up
  • Strong, dynamic typing

              #!/usr/bin/env python
              "7" + 3 == 10  # Raises an exception

Lists, Tuples, & Arrays

PHP

It's mostly about functions... mostly.


  $breakfast = array("eggs", "spam", "bacon", "yogurt", "toast");

  $breakfast[0];  # eggs
  $breakfast[2];  # bacon

  array_slice($breakfast, -1);      # "toast"
  array_slice($breakfast, -2, 2);   # ["yogurt", "toast"]
  array_push($breakfast, "juice");  # Adds "juice" to the array
  array_pop($breakfast);            # Pops "juice" off the array

  $united_kingdom = array_merge(
      array("England", "Wales"),
      array("Scotland", "Northern Ireland")
  );  # Returns an array of the UK

Python Lists

Everything is an object!


  breakfast = ["eggs", "spam", "bacon", "yogurt", "toast"]

  breakfast[0]               # eggs
  breakfast[2]               # bacon

  breakfast[-1]              # toast
  breakfast[-2:]             # ["yogurt", "toast"]
  breakfast.append("juice")  # Adds "juice" to the list
  breakfast.pop()            # Pops "juice" off the list

  united_kingdom = ["England", "Wales"] + ["Scotland",
      "Northern Ireland"]  # Returns an array of the UK

Objects

PHP

Objects were a late-to-the-party addition


  class Cake {
      public $flavour;
      public $icing;
      private $temperature;
      function __construct($flavour, $icing) {
          $this->flavour = $flavour;
          $this->icing = $icing;
          $this->temperature = 22;
      }
      public function bake(){
          while ($this->temperature < 180) {
              $this->temperature++;
          }
      }
  }

Python

Objects were part of Python from the beginning


  class Cake(object):

      def __init__(self, flavour, icing):
          self.flavour = flavour
          self.icing = icing
          self._temperature = 22

      def bake(self):
          while self._temperature < 180:
              self._temperature += 1

Python

Objects are magical!

Packaging and Namespacing

PHP

Use of include or require simply injects that code into the running environment. 


  # variables.php
  $favourite_colour = "blue";
  $favourite_food = "cake";

  # myscript.php
  require("variables.php");
  print $favourite_food;  # "cake"

Python

Packages are isolated from each other


  # variables.py
  favourite_colour = "blue"
  favourite_food = "cake"

  # myscript.py
  import variables
  print(favourtie_food)  # Exception!
  print(variables.favourite_food)  # "cake"

  # myotherscript.py
  from variables import favourite_colour
  print(favourite_colour)  # "blue"
